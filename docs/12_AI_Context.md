# Project Information

## Project Name

Orphanage Management System (OMS)

---

## Project Type

Enterprise Web Application

---

## Project Goal

Develop a secure, scalable, maintainable, and cost-effective web application for managing students within an orphanage.

The application will manage the complete lifecycle of a student from admission until they leave the orphanage while preserving historical records.

The application replaces paper-based record management with a secure digital platform.

---

# Primary Objectives

The application must:

- Digitize student records.
- Simplify student enrollment.
- Store student documents securely.
- Maintain historical records.
- Generate printable reports.
- Be easy for orphanage staff to use.
- Follow enterprise architecture.
- Be production ready.
- Be cloud deployable.
- Keep hosting costs as low as possible.

# Soft Delete Strategy

Student records must never be physically deleted.

Use

deleted

deletedDate

deletedBy

status = INACTIVE

Default queries must exclude deleted records.

Administrators may retrieve historical records.
