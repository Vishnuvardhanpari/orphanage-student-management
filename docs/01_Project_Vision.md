# Project Vision

# Orphanage Management System (OMS)

## Overview

The Orphanage Management System (OMS) is a secure, modern, web-based application designed to digitally manage student records within an orphanage.

The application replaces manual and paper-based record management with a centralized, secure, and searchable system that stores student information, educational records, personal details, scanned documents, and historical records.

The system manages the complete lifecycle of every student—from enrollment until they leave the orphanage—while permanently preserving historical records using Soft Delete instead of physical deletion.

The application is intended to be simple enough for orphanage staff to use while following enterprise-grade software engineering practices.

---

# Objectives

The primary objectives of the application are:

- Digitize all student records.
- Eliminate paper-based record keeping.
- Simplify student enrollment.
- Securely store student documents.
- Maintain complete historical records.
- Prevent accidental deletion of records.
- Provide powerful search and filtering.
- Generate printable student reports.
- Minimize hosting and maintenance costs.
- Build a scalable and maintainable application.

# Student Management

Student Management is the core module of the application.

Each student record contains:

- Personal Information
- Guardian Information
- Educational Information
- Medical Information
- Admission Information
- Exit Information
- Student Photo
- Multiple Uploaded Documents

Documents are managed as part of the Student Profile.

There is no separate Document Management module.

---

# Reports Module

The Reports module allows users to:

- Export a single student.
- Export multiple students.
- Export filtered student records.
- Generate printable PDF reports.

Reports should include:

- Student Photograph
- Student Details
- Uploaded Document Images (where applicable)
- Document Information
- Generated Date
- Generated By

---

# Historical Records

Student records must never be permanently deleted.

When a student leaves the orphanage:

- Mark the record as inactive.
- Soft Delete the record.
- Preserve all uploaded documents.
- Preserve audit history.

Historical records should remain searchable by administrators.
