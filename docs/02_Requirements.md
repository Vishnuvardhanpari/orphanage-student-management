# Functional Requirements

# Authentication

The application shall support:

- Username & Password Login
- Google Gmail Login
- JWT Authentication
- Secure Logout
- Password Encryption using BCrypt
- Session Expiry
- Refresh Token Support (Future)

---

# User Management

Administrator can:

- Create Staff Accounts
- Disable Accounts
- Enable Accounts
- Reset Password
- Assign Roles

Staff cannot manage users.

---

# Student Management

The application shall allow users to:

- Add Student
- View Student
- Edit Student
- Soft Delete Student
- Restore Student (Admin Only)
- Search Student
- Filter Students
- View Active Students
- View Inactive Students

---

# Student Enrollment

The application shall capture:

## Personal Information

- Admission Number
- First Name
- Last Name
- Gender
- Date of Birth
- Blood Group
- Religion
- Nationality
- Aadhaar Number (Optional)
- Contact Number

---

## Guardian Information

- Guardian Name
- Relationship
- Mobile Number
- Address

---

## Educational Information

- School Name
- Standard
- Medium
- Previous School
- Admission Date

---

## Medical Information

- Medical Conditions
- Allergies
- Disability
- Emergency Notes

---

## Exit Information

- Exit Date
- Exit Reason
- Remarks

---

# Student Documents

Documents are uploaded while creating or editing a student.

There is no standalone Document Management module.

Each student can have multiple documents.

Supported documents include:

- Student Photograph
- Aadhaar Card
- Birth Certificate
- Transfer Certificate
- Mark Sheets
- Medical Reports
- Identity Proof
- Other Supporting Documents

Supported Formats

- PDF
- JPG
- JPEG
- PNG

Maximum File Size

10 MB

Documents should be:

- Uploaded
- Previewed
- Downloaded
- Replaced
- Deleted (Logical removal if required)

---

# Search

Users should search students using:

- Admission Number
- Name
- Aadhaar Number
- Mobile Number
- Status
- Admission Date
- Leaving Date

---

# Filters

Users should filter students by:

- Gender
- Status
- Admission Year
- Leaving Year
- Age
- School
- Created Date

---

# Dashboard

Dashboard should display:

- Total Students
- Active Students
- Left Students
- New Admissions
- Recent Updates

---

# Reports & Export

This is a dedicated module.

Users can:

Export Single Student

Export Multiple Students

Export Filtered Students

Generate PDF

Reports should include:

- Student Photograph
- Student Information
- Uploaded Documents
- Generated Date
- Generated By

Future enhancements:

- Excel Export
- QR Code
- Barcode

---

# Audit

Every important action shall be audited.

Examples:

- Student Created
- Student Updated
- Student Deleted
- Student Restored
- Document Uploaded
- Document Replaced
- User Login
- User Logout

---

# Non Functional Requirements

## Performance

- Fast Search
- Pagination
- Optimized Queries

## Security

- JWT Authentication
- Google OAuth2
- BCrypt Password
- Role Based Access
- HTTPS

## Scalability

Support thousands of students.

## Availability

99% uptime preferred.

## Maintainability

Follow Clean Architecture.

## Cost

Infrastructure should be optimized for minimum operational cost.
